# Стиль текста вики PRDX

## Обращение

На "ты" везде. Не "вы", не "Вы". Императив: "наведи", "напиши", "нажми". Притяжательные: "твой", "твои", не "ваш"

## Тон

Дружелюбный, конкретный, без воды. Не разговорный ("мне лень писать гайд"), не казённый ("Уважаемый пользователь"). Факт → пояснение → пример. Не извиняться, не оправдываться

## Точки

Точка в конце строки не ставится. Точки только между предложениями внутри одного абзаца

## Структура страницы

- Под `# Заголовок` — одно-два предложения о чём эта штука и зачем. Без перечисления фич, они и так ниже
- Секции через `## H2`, каждая самодостаточна
- Команды в `CommandTableFilter` когда их больше 2-3, иначе инлайн-код
- Клавиши через `:kbd[...]`

## Предложения

- Короткие. Одна мысль — одно предложение
- Императив для инструкций: "Наведи", "Напиши", "Открой"
- Тире `—` использовать редко, когда действительно нужно

## Запрещено

- "Это не просто X — это Y"
- "Давайте разберёмся" / "Давайте погрузимся"
- "Стоит отметить, что"
- Восторженные прилагательные без повода: "уникальный", "невероятный", "революционный"
- Перечислять фичи в описании если они и так перечислены на странице ниже
- Начинать с широкого пафосного утверждения
- Секции "Итоги" / "Ключевые выводы" / "Подведём итог"
- Риторические вопросы как переходы между секциями
- Хеджирование: "возможно", "потенциально", "время покажет"

## Жаргон

Термины вроде "X-Ray", "макросы", "NoClip" — пояснять при первом упоминании если страница для новичков. На страницах для опытных (creative, rules) можно без пояснений

## Компоненты

Доступные компоненты описаны в `README.md`. Основные:
- `CustomBlock` — алерты (note/tip/important/warn/danger)
- `Collapse` / `CollapseBlock` — спойлеры
- `FatLink` — кнопки-ссылки
- `Badge` / `BadgeGroup` — цветные метки
- `CraftGrid` — рецепты крафта
- `CommandTableFilter` — таблица команд с поиском
- `AnimatedMedia` — видео/анимации
- `InlineSpoiler` — скрытый текст
- `Grid` — сетка изображений
- `:kbd[...]` — клавиши
- `figure` + `:figcaption[...]` — картинка с подписью
