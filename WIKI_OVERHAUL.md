2. **Основные команды**

| Команда | Синтаксис | Описание | Пример | Видимость |
|---------|-----------|---------|--------|-----------|
| `/me` | `/me <действие>` | Ваше действие | `/me наклоняется и кидает камень` | Local (128 блоков) |

   - Пишите от третьего лица: "делает X", "видит Y"
   - Не спамьте командами, используйте их в RP ситуациях

4. **Примеры хорошего использования**
   ```
   /me достаёт карту из рюкзака и изучает её
   /try пытаюсь открыть запертую дверь
   /do магический символ светится и исчезает
   /gme всё небо закрывается тучами
   ```

5. **Примеры плохого использования**

### 4.2 Рекомендуемый стиль гайда

**Тон:** Дружелюбный, но информативный. Обращаться к игроку как к коллеге, но объяснять сложные концепции.

**Структура абзаца:**
- Начните с **главной идеи**
- Дайте **контекст** (зачем это нужно)
- Объясните **как это работает**
- Дайте **пример** или **совет**

**Пример неправильного текста:**
```
Это команда для сидения. /sit это здорово для RP.
```

**Пример правильного текста:**
```
Команда `/sit` позволяет вашему персонажу сидеть, что полезно для RP ситуаций (переговоры, отдых, и т.д.).
Просто введите `/sit` и персонаж сядет на землю или на стул, если он рядом.
Повторный ввод `/sit` поднимет персонажа.
```

**Правила консистентности:**
1. Используйте **`/command`** для команд (с обратным апострофом)
2. Используйте **`item_name`** для названий предметов (курсив или code)
3. Используйте **Y: число** для координат высоты
4. Используйте **100 блоков** или **128 блоков** (с числом)
5. Используйте **RMB** для правой кнопки мыши, **LMB** для левой
6. Используйте **[IN DEV]** для незаконченных функций

### 5.1 Текущие MDX компоненты

Компоненты, которые используются в текущей вики:

```
<CollapseBlock> — свёртываемый блок текста
<Collapse> — альтернативный свёртываемый компонент (нужно проверить разницу)
<CustomBlock> — пользовательский блок (цвет, фон)
<FatLink> — большая ссылка-кнопка
<Grid> — сетка для расположения элементов
<figure> — для изображений
:kbd[] — для отображения клавиш (e.g., `:kbd[V]`)
:figcaption[] — подпись к изображению
```

### 5.2 Нужные новые компоненты

**CraftingRecipe — Визуальный рецепт крафтинга**

Использование:
```mdx
<CraftingRecipe
  name="Деревянная мотыга"
  pattern={[
    ["wood", "wood", null],
    [null, "stick", null],
    [null, "stick", null],
  ]}
  result={{ name: "stick_recipe", amount: 1 }}
/>
```

Рендеринг: 3x3 сетка как в игре, с результатом справа.

**Alert / Warning — Баннер оповещения**

Использование:
```mdx
<Alert type="warning" title="ФУНКЦИЯ В РАЗРАБОТКЕ">
  Эта функция будет доступна позже в сезоне.
</Alert>

<Alert type="info" title="СОВЕТ">
  Используйте этот совет для лучшего результата.
</Alert>

<Alert type="error" title="ВНИМАНИЕ">
  Это может привести к потере данных!
</Alert>
```

Стили: warning (жёлтый), info (синий), error (красный), success (зелёный)

**CommandTable — Таблица команд**

Использование:
```mdx
<CommandTable
  commands={[
    { command: "/home", description: "Вернуться в дом", example: "/home" },
    { command: "/spawn", description: "К спавну", example: "/spawn" },
  ]}
/>
```

Рендеринг: таблица с специальной стилизацией для команд (моноширинный шрифт).

**BadgeGroup — Группа иконок/бейджей**

Использование:
```mdx
<BadgeGroup
  items={[
    { label: "[IN DEV]", color: "yellow" },
    { label: "Новое", color: "green" },
  ]}
/>
```

Используется для маркирования статуса функций.

**ExpandableSection — Раскрываемая секция**

Использование:
```mdx
<ExpandableSection title="Секрет: как спрятаться">
  Текст, который скрыт по умолчанию...
</ExpandableSection>
```

Альтернатива CollapseBlock с улучшенным UX.
